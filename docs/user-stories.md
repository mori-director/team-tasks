# 유저 스토리 — team-tasks MVP

## 1. 핵심 시나리오 — 페르소나 1 (일반 직원 · 개발자 민준)

> 로그인 없이 브라우저 북마크 하나로 접속하며, 하루 동안 team-tasks를 어떻게 사용하는지 시간 순서로 기술합니다.

| 시간 | 행동 | 앱에서 하는 일 | 기대 결과 |
|------|------|----------------|-----------|
| **09:00** 출근 직후 | 오늘 할 일 파악 | 보드를 열고 담당자 필터에서 **내 이름(민준)** 을 선택한다 | '할 일' 컬럼에 나에게 배정된 카드만 표시된다 |
| **09:10** 우선순위 정리 | 긴급 업무 식별 | 우선순위 필터를 **높음**으로 바꿔 오늘 반드시 처리해야 할 카드를 확인한다 | 높음 우선순위 카드 N개만 보인다 |
| **09:20** 첫 업무 착수 | 상태 변경 | 가장 시급한 카드를 클릭해 상태를 **'할 일' → '진행 중'** 으로 이동한다 | 카드가 '진행 중' 컬럼으로 이동되어 팀 전체에 착수 사실이 보인다 |
| **11:30** 중간 점검 | 진행 상황 메모 | 카드 수정 다이얼로그를 열어 설명란에 **현재 진척 내용과 이슈**를 자유 텍스트로 남긴다 | 설명이 저장되어 팀장이 별도 문의 없이 상황을 파악할 수 있다 |
| **13:00** 점심 후 복귀 | 신규 배정 확인 | 필터를 초기화하고 전체 보드를 훑어 점심 사이 **새로 배정된 카드**가 있는지 확인한다 | 팀장이 추가한 새 카드가 '할 일' 컬럼에 보인다 |
| **14:00** 오후 업무 착수 | 두 번째 일감 시작 | 오후 일감 카드를 **'진행 중'** 으로 이동한다 | 현재 진행 중인 카드가 2개인 상태가 보드에 반영된다 |
| **16:30** 첫 번째 일감 완료 | 상태 변경 + 링크 기록 | 오전 일감 카드를 **'완료'** 로 이동하고 설명란에 PR 링크를 남긴다 | '완료' 컬럼으로 카드가 이동되고 링크가 저장된다 |
| **18:00** 퇴근 전 정리 | 하루 마무리 확인 | 담당자 필터로 내 카드만 다시 조회해 미완료 카드의 **마감일과 우선순위**를 점검한다 | 내일 이어서 할 카드 목록과 마감일이 한눈에 보인다 |

### 이 시나리오가 전제하는 MVP 기능

1. 담당자 필터 (내 이름으로 카드 좁혀 보기)
2. 우선순위 필터 (높음·보통·낮음)
3. 카드 상태 이동 (할 일 → 진행 중 → 완료)
4. 카드 설명 자유 텍스트 편집
5. 마감일 표시 (지남 여부 강조)

## 2. 결정 사항

페르소나 시나리오에서 도출한 미결 질문 5개입니다. 답변에 따라 기능 범위가 달라집니다.

1. **누구나 일감을 만들 수 있는가?**
   팀장(P1)만 생성 권한을 갖는가, 아니면 담당자(P2)와 협력사(P3)도 직접 일감을 추가할 수 있는가?

2. **담당자도 다른 사람에게 일감을 넘길 수 있는가?**
   배정된 일감의 담당자 필드를 본인이 수정할 수 있는가, 아니면 팀장만 재배분 가능한가?

3. **협력사(P3)는 자기 카드를 어떻게 찾는가?**
   담당자 필터를 직접 선택하는 것으로 충분한가, 아니면 `?assignee=수아` 같은 전용 URL이 있어야 하는가?

4. **설명란은 자유 텍스트인가, 구조화된 필드인가?**
   PR 링크·메모를 하나의 텍스트 박스에 자유롭게 입력하는가, 아니면 링크 전용 필드를 별도로 둘 것인가?

5. **완료된 일감은 어떻게 처리하는가?**
   '완료' 컬럼에 계속 쌓아 두는가, 일정 기간 후 숨기거나 별도 아카이브 뷰로 이동하는가?
