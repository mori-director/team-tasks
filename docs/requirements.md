# 요건 정의 — team-tasks MVP

> 출처: `docs/personas.md` · `docs/user-stories.md`
> MVP 헌장 준수: 기능 요건 5개 이하, Phase 2 항목 제외.

---

## 1. 기능 요건

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|---------|
| F-01 | 일감 CRUD | 제목·설명·담당자·마감일·우선순위를 포함한 일감을 생성·조회·수정·삭제한다. 로그인 없이 누구나 조작 가능. | 필수 |
| F-02 | 칸반 보드 | 할 일 / 진행 중 / 완료 3개 컬럼을 한 화면에 표시한다. 카드 드롭다운으로 컬럼 간 이동(상태 변경)이 가능하다. | 필수 |
| F-03 | 담당자·우선순위 필터 | 담당자 또는 우선순위 기준으로 카드를 즉시 좁혀 볼 수 있다. 필터 값은 `localStorage`에 저장해 새로고침 후에도 유지된다. | 필수 |
| F-04 | 마감일 초과 강조 | 오늘 날짜를 초과한 미완료 카드의 날짜를 빨간색으로 표시한다. 추가 알림(뱃지·고정)은 MVP 범위 외. | 필수 |
| F-05 | 설명 자유 텍스트 편집 | 카드 설명란을 자유 텍스트로 수정·저장한다. 수정 시 이전 내용은 덮어쓰며, 이력 관리는 MVP 범위 외. | 필수 |

## 2. 비기능 요건

### 성능
| 항목 | 기준 |
|------|------|
| 초기 페이지 로드 | 카드 100개 기준 LCP 2 초 이하 (3G 환경 제외) |
| 상태 변경 반응 | 카드 이동 후 UI 반영 200 ms 이하 (낙관적 업데이트 적용) |

### 보안
| 항목 | 기준 |
|------|------|
| 입력 검증 | 모든 API 입력값을 서버에서 허용 타입·길이로 검증, SQL Injection·XSS 차단 |
| 민감 정보 노출 | 응답 바디에 서버 내부 경로·스택 트레이스를 포함하지 않음 |

### 가용성
| 항목 | 기준 |
|------|------|
| 서비스 업타임 | 배포 환경(Vercel 등) SLA 기준 월 99.5 % 이상 |
| 데이터 지속성 | 서버 재시작 후에도 `data/tasks.json` 내 일감 유실 0건 |

### 접근성
| 항목 | 기준 |
|------|------|
| 키보드 탐색 | 모달·드롭다운 포함 모든 인터랙션을 키보드만으로 완료 가능 (Tab · Enter · Esc) |
| 색상 대비 | WCAG 2.1 AA 기준 — 텍스트 대비율 4.5 : 1 이상 (마감 초과 빨간 텍스트 포함) |
